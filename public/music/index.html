网络设置 (Network Settings)

播放音乐代理 (Proxy Playback) 通过服务器端代理解析和播放音乐，有效解决部分源的 IP 限制问题。
下载音乐代理 (Proxy Download) 通过服务器端代理执行音乐下载操作。
自动代理 (Auto Proxy) 仅在 HTTPS 浏览器环境下，获取到非安全 (HTTP) 链接时自动检测升级HTTPS或启用代理解析，确保播放成功。
自定义代理 (Custom Proxy) 开启后，当自动代理触发时，将优先使用您填写的自定义代理地址（客户端直接请求），而非服务器中转。
URL 格式示例：https://proxy.example.com/{url}
其中 {url} 将被替换为原始音乐链接（不编码）。需同时开启"自动代理"才生效。

使用 {url} 作为原始链接占位符，将被直接替换为原始音乐 URL（不编码）。

缓存并发量 (Concurrency) 控制本地与云端同时执行的任务数量（建议 3-5）。

自动更新 (Auto Update)

自动更新网络歌单 (Auto Update List) 开启后，点击进入网络歌单时将自动尝试同步远端内容。注意：此操作可能会覆盖本地对该歌单的修改。
自动同步账号设置到文件 (Auto-sync settings to file) 保存当前账号的设置（音质、源选择等）到服务器端用户配置目录，实现多端自动同步。注意：未登录状态下，该设置无效，账号设置将保存在浏览器缓存。

手动将当前所有 UI 设置（包含音质、音效等）缓存并保存至服务器对应用户的对应json文件中。

本地缓存与重置 (Cache & Reset)

缓存歌词 (Cache Lyrics) 开启后，应用会自动缓存已加载的歌词。
缓存歌曲链接 (Cache Song URL) 优先使用缓存链接播放，失效时系统会自动尝试重新获取并更新缓存。
缓存歌曲文件 (Cache Song File) 将歌曲文件永久缓存到服务器存储。再次播放时无需消耗外部流量，极大提升加载速度。
缓存歌词文件 (Cache Lyric File) 将歌词文件缓存到同服务器配置的缓存目录下，优先供多端同步极速加载。
优先播放缓存歌曲 (Prefer Cached Song) 如果歌曲已缓存，则优先从本地加载。即使“缓存歌曲文件”关闭，已存在的缓存仍可使用。
仅下载模式 (Only Download Mode) 未开启时下载到 /server/cache/用户名 (或 /server/data/cache/)；
开启后下载到 /server/music/用户名 (或 /server/data/music/)。
缓存歌曲位置 (Cache Location) 设置服务器端存储缓存文件的具体路径。
歌曲命名方式 (Naming) 设置服务端存储文件的命名规则（对标准缓存和仅下载模式均生效）。不影响已有文件。
优先级: 本地文件 > 链接缓存 > 在线获取 (自动缓存)

播放逻辑 (Playback Logic)

播放搜索歌曲时切换歌单 (Switch Play Context) 开启：播放搜索出的歌曲后，播放列表切换为搜索结果。
关闭：播放搜索出的歌曲后，播放列表切换为默认列表。
* 提示：所有搜索到的歌曲播放时都会自动加入默认列表

本地列表播放去重 (Deduplicate Local List) 当播放本地音乐（列表）时，若存在同 ID 但音质不同的歌曲文件，仅保留最高音质版本。说明有相同id歌曲，只播放最高音质歌曲。

播放歌单/排行榜歌曲时切换歌单 (Switch Song List / Leaderboard Play Context) 开启：播放歌单或排行榜中的单个歌曲后，播放列表切换为对应列表。
关闭：播放歌单或排行榜中的单个歌曲后，保持当前播放队列不变。
* 提示：所有歌单/排行榜歌曲播放时会自动加入默认列表

音乐预读机制 (Pre-fetching) 开启：在一首歌曲开始播放时，系统将自动在后台预解析并读取下一首歌曲的播放内容。预读过程中会自动跳过无法播放的歌曲。
* 极大提升移动端后台播放的连续性，但可能会产生少量额外流量。

自动恢复进度 (Auto-Resume) 自动恢复最后播放的歌曲、进度及列表。会自动跳转到上次对应的歌单选项卡。

音频淡入淡出 (Crossfade) 切换歌曲时，音量平滑淡出和淡入，减少切换的突兀感。

保持屏幕常亮 (Keep Screen Awake) 防止由于系统自动息屏导致的播放中断，兼容绝大多数移动端设备。

自动尝试换源 (Auto-Switch Source) 当前音质及其所有降级音质解析失败后，根据“歌名+歌手+时长”在全网精准匹配同名歌曲尝试播放。
* 开启：自动搜索备选音源 | 关闭：原始源失效即报错

失败自动下一曲 (Auto-Skip on Error) 当前歌曲播放失败3s后自动播放下一曲。
* 开启：自动尝试下一首 | 关闭：暂停播放

SMTC 歌词显示 (SMTC Lyric Display) 将当前播放的歌词行实时写入系统媒体控制 (SMTC/MediaSession) 的标题字段，使 Windows 任务栏、系统通知中心等能同步显示当前歌词。
* 原理：利用 Web Media Session API 高频更新 metadata.title 为当前歌词行，标题会随歌词进度自动滚动。
关闭后恢复显示原始歌曲名称。
